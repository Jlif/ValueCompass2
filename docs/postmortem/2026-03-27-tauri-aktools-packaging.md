**日期**: 2026-03-27
**问题**: Tauri 应用打包后 Python 服务启动失败、端口冲突、服务检测不准确
**影响**: 应用无法正常使用 AKTools 数据服务
**解决时间**: 约 3 小时

## 时间线

- 13:00 - 开始打包，PyInstaller 打包 Python 服务
- 13:30 - 首次测试，应用启动但 Python 服务失败
- 13:45 - 发现导入错误，`get_default_application` 不存在
- 14:00 - 修复入口脚本，改用动态导入 `aktools.main`
- 14:15 - 端口冲突，8080 被微信占用
- 14:30 - 实现动态端口分配（18080 起）
- 14:45 - 服务检测失败，不断启动新服务
- 15:00 - 发现健康检查端点问题，改用 `/version`
- 15:15 - 按钮状态问题，添加禁用状态
- 15:30 - 服务停止问题，通过 lsof/pkill 停止进程

## 根因分析

### 问题 1: Python 服务启动失败

**表面原因**: `get_default_application` 导入失败
**真正原因**: AKTools 版本变更，旧 API 已不存在

### 问题 2: 端口冲突

**表面原因**: 8080 被占用
**真正原因**: 没有实现端口自动发现和冲突处理

### 问题 3: 重复启动服务

**表面原因**: `is_running()` 返回 false
**真正原因**:
1. 根路径 `/` 返回 500，健康检查失败
2. 检测到已存在服务时没有保存进程句柄

### 问题 4: 服务无法停止

**表面原因**: 点击停止按钮无效
**真正原因**: 没有进程句柄，需要通过系统命令停止

## 教训

1. **外部服务检测要选对端点**
   - AKTools 的 `/` 返回 500，但 `/version` 返回 200
   - 健康检查应该调用可靠的 API 端点

2. **动态端口分配要考虑多种情况**
   - 端口被占用
   - 服务已存在但未记录
   - 服务自己启动的需要能停止

3. **进程管理要保存句柄**
   - 检测到已存在的服务时也要记录 PID
   - 否则无法优雅停止

4. **前端按钮要考虑状态流转**
   - Starting 状态要禁用按钮防止重复点击
   - Running/Stopped 状态要有明确区分
