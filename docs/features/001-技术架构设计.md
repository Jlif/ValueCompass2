## 2. 数据存储策略

### 2.1 本地持久化存储（SQLite）

仅存储轻量级元数据，不存储海量 K 线：

```sql
-- 股票基础信息表 (~5000 条)
CREATE TABLE stocks (
    code TEXT PRIMARY KEY,           -- 股票代码，如 600519
    name TEXT NOT NULL,              -- 股票名称
    exchange TEXT,                   -- 交易所 SH/SZ/BJ
    industry TEXT,                   -- 所属行业
    market_cap REAL,                 -- 总市值
    list_date DATE,                  -- 上市日期
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 用户自选表
CREATE TABLE watchlist (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    code TEXT NOT NULL,
    added_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (code) REFERENCES stocks(code),
    UNIQUE(code)
);

-- 数据同步日志
CREATE TABLE sync_log (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    table_name TEXT NOT NULL,
    sync_type TEXT,                  -- full/incremental
    records_count INTEGER,
    started_at TIMESTAMP,
    completed_at TIMESTAMP,
    status TEXT,                     -- success/failed
    error_message TEXT
);
```

## 3. 服务生命周期管理

### 3.1 Python 服务启动

```rust
// src-tauri/src/python_service.rs
use std::process::{Child, Command};
use std::sync::Mutex;

pub struct PythonService {
    process: Mutex<Option<Child>>,
    port: u16,
}

impl PythonService {
    pub fn start(&self) -> Result<(), String> {
        let executable = get_aktools_path()?;
        let child = Command::new(executable)
            .arg(format!("--port={}", self.port))
            .spawn()
            .map_err(|e| format!("Failed to start AKTools: {}", e))?;

        *self.process.lock().unwrap() = Some(child);

        // 等待服务就绪
        wait_for_service(self.port)?;
        Ok(())
    }
}
```

### 3.2 应用生命周期

```
用户打开 App
    ↓
Tauri 启动
    ↓
启动 Python 服务 (AKTools)
    ↓
等待服务就绪 (健康检查)

    ↓

用户关闭 App
    ↓
Tauri 关闭事件
    ↓

    ↓

    ↓
退出
```

## 4. 关键技术决策

### 4.1 为什么选 Tauri 而非 Electron

| 对比维度 | Tauri | Electron |
|---------|-------|----------|
| 安装包大小 | ~5-10MB | ~150-200MB |
| 内存占用 | ~100MB | ~300-500MB |
| 启动速度 | 快 | 较慢 |
| 安全模型 | 严格沙箱 | 较宽松 |
| 系统集成 | Rust 原生 API | 依赖 Node 模块 |
| 跨平台 | 支持 | 支持 |

### 4.2 为什么用 AKTools 而非直连

- **开发效率**：AKTools 已封装 AKShare 所有接口，无需重复开发
- **维护成本**：AKShare 团队维护接口适配，跟随数据源变更
- **接口稳定**：HTTP API 比直接调用 Python 函数更易调试

### 4.3 为什么不存全量 K 线

| 方案 | 数据量 | 优缺点 |
|------|--------|--------|
| 存全量 K 线 | ~2-3GB | 离线可用，但首次同步慢，占用大 |
| 只存列表 | ~1MB | 轻量快速，依赖网络 |
| **选中方案**：按需拉取 + 内存缓存 | ~50MB(缓存) | 平衡方案，满足日常使用 |

## 6. 性能考量

### 6.1 启动优化

- **冷启动**：Tauri 应用启动 < 500ms
- **服务启动**：Python 服务启动 2-5s，需显示加载状态
- **首屏渲染**：本地 SQLite 读取 < 100ms

### 6.2 数据加载优化

- **防抖处理**：股票搜索输入防抖 300ms
- **分页加载**：K 线数据分页，避免一次性加载过多
- **缓存预热**：启动时预加载自选股票数据

### 6.3 内存管理

- **缓存上限**：内存最多缓存 50 只股票 K 线
- **LRU 淘汰**：超出上限时淘汰最久未访问的数据
- **定期清理**：每 30 分钟清理过期缓存

## 7. 技术风险与缓解

| 风险 | 影响 | 缓解方案 |
|------|------|----------|
| AKTools 启动失败 | 高 | 检测端口占用，自动重试，友好错误提示 |
| 东方财富限流 | 中 | 请求防抖，错误重试，指数退避 |
| 网络不可用 | 中 | 离线模式提示，显示缓存数据 |
| Python 打包兼容性 | 中 | PyInstaller 多平台测试，CI 自动构建 |
| SQLite 并发写入 | 低 | Rust 端序列化写入操作 |
